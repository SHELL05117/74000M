# 74000M 真机参数、调试与能力解锁清单

> 用途：机器人机械和电气装配完成后，按本清单把离线实现转为可上机、可驾驶、可自动和可比赛的配置。本文件只记录项目执行顺序与证据要求；平台事实和算法仍以 `C:\Users\alexh\Documents\VEX0713` 中当前 `docs2/`、`docs/` 为准。

## 1. 状态与记录规则

每个参数、测试和配置必须绑定：

- `robot_id`、硬件 revision、配置 schema、校准 revision；
- PROS、VEXos 和软件版本或 commit；
- 数据集/日志 ID、测试日期、场地、轮胎、传动、电池与负载；
- 参数来源：`Assumed`、`Nominal/CAD`、`Measured`、`Fitted`；
- 验证等级：`Unverified`、`PCValidated`、`SimValidated`、`HILValidated`、`FieldValidated`、`CompetitionApproved`；
- 结论：`PASS`、`FAIL`、`REPEAT`、`NOT TESTED`；
- 原值、新值、回滚值和批准人。

示例值、CAD值和离线估计值不得直接升级为比赛参数。一次测试不能同时用于拟合和最终验证。

## 2. 能力默认状态

机器人首次完工时保持：

```text
hardware_output = false
driver_control = false
pose_good = false
autonomous_chassis_velocity = false
autonomous_motion = false
competition_routes = false
selected_route = DoNothing
```

能力必须按本文件顺序逐级解锁。解锁失败时回退到上一已验证状态，不能跳级。

## 3. 第一阶段：机械、电气和设备事实

### 3.1 必须确认的硬件参数

| 项目 | 装车前允许值 | 真机确认方法 | 结果用途 |
|---|---|---|---|
| 左右驱动 Smart Port | 计划端口 | 对照接线表并逐端口识别 | `HardwareConfig` |
| 每个电机反向 | 计划/未知 | 架空单电机正向低压点动 | HAL 单点反向 |
| 电机 cartridge | 铭牌/计划值 | 设备身份与实物检查 | 转速上界和配置校验 |
| 电机数量与左右归属 | 计划值 | 实物、日志和逐端口检查 | 逐电机遥测布局 |
| IMU 端口和安装方向 | 计划值 | 端口识别、人工 CCW 旋转 | 航向正号和安装变换 |
| Rotation Sensor 端口/反向 | 计划值 | 人工正向转轮 | tracking wheel 正号 |
| 驱动轮型号和名义直径 | CAD/型号 | 实物测量并记录磨损/压缩 | 名义几何起点 |
| 传动关系 | CAD/齿数 | 数齿、标记转数检查 | 每个 `MotorPortConfig::motor_rev_per_wheel_rev` |
| 名义轮距 | CAD/估计 | 安装完成后实测 | 有效轮距初值 |
| tracking wheel 半径和外参 | CAD/估计 | 实测接触点与测量轴 | 里程计初值 |
| robot ID / hardware revision | 计划值 | 与实机标签核对 | 配置和日志身份 |

### 3.2 必须执行的检查

1. 检查轮、轴、齿轮、链条、轴承、紧固件和线缆无干涉或异常阻力。
2. 验证 Smart Port 合法、唯一且设备类型正确。
3. 架空并提供物理急停；逐电机施加受限、短时、正向测试请求。
4. 确认“正电压 → 正编码器 → 该侧前进趋势”。反向只在 HAL/静态配置中定义一次。
5. 两侧同给正请求时检查整车前进趋势；发现机械对抗立即锁存停止。
6. 人工逆时针旋转机器人，确认 IMU 累计角增加。
7. 人工正向转动每个 tracking wheel，确认核心 SI 值符号正确。
8. 分别验证 `Coast`、`Brake`、`Hold` 由 `brake()` 路径产生可区分行为。
9. 拔除或模拟一个设备，确认故障能定位到具体端口。

### 3.3 解锁条件

只有全部检查通过，才允许：

```text
hardware_output = true  // 仍只允许受限 Test 请求
```

此时不得开启驾驶或自动。

## 4. 第二阶段：运行时、时序和看门狗

### 4.1 需要真机确定的参数

| 参数 | 离线状态 | 真机调试内容 |
|---|---|---|
| 控制周期 | 10 ms 建议起点 | 测 `raw_dt`、执行时间、jitter、连续超期 |
| OutputService 周期/错相 | 候选值 | 测实际输出年龄和任务竞争 |
| 请求 TTL | 候选值 | 暂停请求源，测误停率与停车延迟 |
| 输出 TTL | 候选值 | 冻结 ControlLoop，测独立停车延迟 |
| 数学 `dt` 上下界 | 仿真值 | 用真机尾延迟确认不掩盖超期 |
| 任务优先级和栈 | 初始配置 | 在最坏 UI/日志/机构负载下测量 |
| 日志容量与转存频率 | PC估计 | SD缺失、慢写、满环和暂停消费者压力测试 |
| HMI刷新频率 | UI候选值 | 验证不改变控制时序 |
| IMU校准超时 | API/候选值 | 多次冷启动和失败注入 |

### 4.2 必须通过的故障注入

- Driver → Disabled → Driver，旧 epoch 满幅帧不能复活；
- 请求停止更新后，请求 TTL 触发停止；
- ControlLoop 停止发布后，输出 TTL 触发配置的 `stop()`/`brake()` 路径；当前 1690X 样机使用 `Coast`；
- 未来时间戳、乱序、NaN/Inf、错误 owner lease 均不能输出；
- Controller 断联后取消驾驶请求、清有记忆状态且不转给自动；
- UI、SD 和日志任务卡住时控制输出和周期预算不变。

通过后才能把运行时参数升级为 `HILValidated`。

## 5. 第三阶段：传感器健康与几何校准

### 5.1 需要采集的健康基线

- 静止时每个电机编码器、IMU、Rotation Sensor 的噪声、重复值和漂移；
- 各设备采集年龄、帧内读取跨度和实际新样本率；
- 正反不同速度下的位置增量、API速度和窗口速度；
- 同侧三电机增量残差和方向一致性；
- IMU rotation 与 gyro rate 的短时一致性；
- 编码器航向与 IMU 航向增量残差；
- 设备断联、冻结、跳变和恢复首帧行为。

### 5.2 需要真机确定的校验参数

| 参数组 | 必须由真机决定的内容 |
|---|---|
| 新鲜度 | 每类样本最大年龄、重复序列容忍和采集跨度预算 |
| 物理范围 | 最大位置增量、最大轮速、IMU角速度和电池合法范围 |
| 同侧一致性 | 绝对残差、相对残差、进入时间和恢复时间 |
| 冻结/跳变 | 幅值、持续时间和交叉证据条件 |
| 速度估计 | 窗口长度、最小时间跨度、低通时间常数和可接受延迟 |
| 质量状态 | Good/Degraded/Invalid 进入与恢复滞回 |
| 短时保持 | 允许沿用旧值的最大时间和用途限制 |

### 5.3 几何标定顺序

1. 正反长直线拟合左、右 `m_per_motor_rad`；训练与验证行程分离。
2. 多次 CW/CCW 原地旋转拟合 `effective_track_width_m`。
3. 若有 tracking wheel，分别拟合有效半径和安装 offset。
4. 执行正/反、CW/CCW 方形或 UMBmark，分离系统误差与随机误差。
5. 验证直线、原地旋转、圆弧、多圈、不同速度和原子重置。
6. 检查滑移时质量会降级；IMU-only 不得发布完整 `Good` 位姿。

满足重复性和质量门后才能设置：

```text
pose_good = true
```

## 6. 第四阶段：基础手动驾驶

### 6.1 需要真机调试的参数

- Controller 各轴固定离线零偏；禁止比赛中在线学习零点；
- throttle/turn deadband；
- 线性与立方权重、端点映射；
- throttle/turn 上升、下降和换向 Slew；
- 曲率增益、Quick Turn 阈值和限制；
- 输出 Slew 与输入 Slew 的分工；
- 当前 1690X 快速候选为 throttle 上升 `20/s`、最终输出上升 `240 V/s`，名义 0→12 V 约 `0.05 s`；旧 `3/s`、`36 V/s` 已收到“加速度偏慢”的 HIL 反馈，新值仍需按正/反、多次重复验证抓地、电流与直线性；
- 零请求的 Coast/Brake 策略；当前 1690X 手动样机统一为 `Coast`，仍需实测自由滑行距离；
- 航向辅助进入/退出速度、杆中立门、`kP/kD` 和最大修正；
- 驾驶档位、最大输出和低电/降额下手感。

### 6.2 验收工况

- 中立无爬行、端点满幅、正反响应近似对称；
- 低速、快速换向、急松杆、原地 Quick Turn；
- 直线全幅仍可按分配策略保留转向；
- 驾驶员转向杆动作同拍释放航向辅助；
- 断控、Disabled 和旧请求均安全停车；
- 8字、推撞和高负载下行为可预测。

通过后才允许：

```text
driver_control = true
```

## 7. 第五阶段：热、电流、卡死、滑移和电池

### 7.1 必须真机基线化的参数

| 类别 | 参数/证据 |
|---|---|
| 温度 | warning、derate、stop、recover、进入/恢复时间、温差阈值 |
| 电流 | 峰值仅记录、窗口均值/RMS、持续高流占空时间 |
| 卡死 | 候选命令电压、低速阈值、确认时间、重复锁存策略 |
| 滑移 | 轮速/机体运动/IMU残差、进入与恢复滞回 |
| 电池 | 静置与负载电压、压降、饱和比例、低裕量策略 |
| 降额 | 各子因子、共同缩放策略、下降/恢复斜率 |
| 停车 | 不同速度、负载下 Coast/Brake/Hold 的停止距离和冲击 |

禁止通过 `12 / battery_V` 放大物理电压请求。低电策略只能限制性能、记录裕量或要求更换电池。

### 7.2 必须覆盖的工况

- 冷态、热态、满电、低电和典型负载；
- 正常加速、Quick Turn、推墙、碰撞候选、轮胎空转；
- 单电机比同侧其他电机更热/高流；
- 保护升级快、恢复慢，Stop/LatchedStop 不能被请求覆盖；
- 清锁只允许 Disabled、原因消失并人工确认。

## 8. 第六阶段：SysId、前馈和轮速闭环

### 8.1 数据采集

按左右侧、正反方向分别执行：

1. quasistatic 电压扫描；
2. dynamic 阶跃；
3. 记录命令 V、实际 V、位置、速度、加速度、电流、温度、电池和质量；
4. 划分训练与验证数据；
5. 拟合并验证 `kS/kV/kA`，不要把一侧或一个方向的结果盲目复制。

### 8.2 需要调试的闭环参数

- 左右侧、正反方向前馈参数；
- 轮速 PID/PI 增益、D低通、积分分离、条件积分和钳位；
- 状态年龄和质量门；
- 反饱和与输出分配；
- settle、timeout、stall、reset；
- 低电、热态和降额时的稳定性。

### 8.3 验收

覆盖阶跃、斜坡、正反、左右分侧、低电和热态。状态陈旧或质量不足时自动请求必须被拒绝。底层冻结门通过后才允许：

```text
autonomous_chassis_velocity = true
```

## 9. 第七阶段：自动运动和路线

### 9.1 自动原语逐级调试

按低速开始，依次执行：

1. BrakeCommand；
2. 短/长、正/反 DriveDistance；
3. 小角/大角、CW/CCW TurnToHeading；
4. DriveArc；
5. DriveToPose；
6. 取消、模式切换、定位降级、timeout、stall 和 interrupted；
7. 低电、热态、负载和扰动复验。

每条命令必须非阻塞，只发布 `DriveRequest`。通过后才能设置 `autonomous_motion=true`。

### 9.2 路径与轨迹

依次验证：

- 起终点、曲率、速度/加速度/角速度/电压/抓地约束；
- 前进、后退和非零初速衔接；
- 直线、圆弧、S弯、8字；
- 状态质量/年龄门、滑移降速/中止；
- 终点 settle、timeout 和扰动恢复；
- 仿真 → 重放 → HIL → 低速真机 → 完整速度。

### 9.3 路线批准

每条路线单独记录 RouteId、起始位姿、机构状态、质量要求、日志、成功率和回滚版本。未实现、未确认、质量不足或未批准的路线一律执行 `DoNothing`。

只有逐条达到 `CompetitionApproved` 后才允许：

```text
competition_routes = true
```

## 10. 第八阶段：HMI、持久化和比赛冻结

必须真机验证：

- Brain 480×240 用户区布局和 Controller 3×19 字符边界；
- Brain/Controller revision 一致；
- 启用后只读、epoch 变化回滚未提交事务；
- 自动 draft/confirm/lock 与 `DoNothing` 回退；
- 参数 staged/apply/rollback，Apply 与 Save 分离；
- SD 双槽、CRC、回读、坏文件、错 robot ID/schema 和掉电恢复；
- renderer 卡住、Controller 断联、事件队列满和 SD 失败不影响控制时序；
- CompetitionApproved 配置、路线、软件版本、配置哈希和回滚包一致。
