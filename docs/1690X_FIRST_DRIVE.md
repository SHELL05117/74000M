# 1690X 样机首次手动试车规程

本规程只适用于当前 `1690X` 六马达样机和 4 V commissioning 构建。它用于确认端口、方向、停车和基础 Arcade 操作，不等于 `hardware_output` 或 `driver_control` 已通过正式验收。

## 1. 已写入程序的事实

- 左侧端口：11 / 12 / 13；右侧端口：1 / 2 / 3；
- 600 RPM 路径：36T → 48T，`motor_rev_per_wheel_rev = 48/36 = 4/3`；
- 200 RPM 路径：36T → 12T，同轴 36T → 48T，`motor_rev_per_wheel_rev = (12/36)×(48/36) = 4/9`；
- 两条路径的理论轮速均为 450 RPM；
- 驱动轮名义直径 2.75 in / 0.06985 m；名义轮距 6 in / 0.1524 m；
- 外部传感器未安装：无 IMU、无 Rotation Sensor；
- 操作：左摇杆 Y 前后、左摇杆 X 转向，单摇杆 Arcade；
- 最大输出 4 V；自动、位姿和航向辅助全部锁定。

## 2. 软件安全行为

- 只有 `OutputService` 写六个底盘马达；
- 请求路径为 `Test → Scheduler/Arbiter → SafetyGate → ActuatorFrame → OutputService`；
- 请求 TTL 为 30 ms，输出帧 TTL 为 40 ms；
- 场控连接时 commissioning Test 强制 Disabled；
- 控制器断联、模式变化、陈旧帧、错误 epoch 或非法数值均 Brake；
- `B` 为锁存停车，当前模式内不能重新解锁；
- `autonomous()` 始终 `DoNothing`。

## 3. 架空首测

准备：

- 底盘可靠架空，所有驱动轮不接触地面；
- 一名操作员持控制器，一名观察员随时按 Brain 停止/禁用；
- 不连接场控；确认电池、齿轮、轴、轮、线缆和紧固件正常；
- 六个轮附近无人、无工具和松散线缆。

步骤：

1. 下载程序，进入 Driver Control。
2. 左摇杆保持中立，同时按住 `L1+L2+R1+R2` 1 秒，然后全部松开。
3. 左 Y 轻推前进约 10% 行程并立即回中；确认六个轮都产生整车前进趋势。
4. 左 Y 轻拉后退；确认六个轮反向且无相互顶死、异响或明显速度失配。
5. 在油门为零时，左 X 轻推右；应出现左侧前进、右侧后退的右转趋势。
6. 按 `B`；所有马达必须立即 Brake。继续推杆不得恢复输出。
7. 禁用再启用，重新执行组合键；确认旧请求不会复活。
8. 受控关闭控制器或断开连接；确认停车，恢复连接后必须重新授权。

任何一步出现方向错误、机械对抗、异常齿声、剧烈振动、线缆干涉、持续高电流或不能停车，立即禁用并停止测试。不要用更高电压“验证能否顶过去”。

## 4. 落地低速测试

只有架空全部通过后才落地：

1. 选择空旷平整地面，周围留足停车距离；仍保持 4 V 上限。
2. 低杆量前进、后退、缓慢左右转弯，检查控制方向和中立无爬行。
3. 做短直线并观察是否存在明显持续偏航；此时只记录，不调 PID。
4. 测试回中、快速松杆和 `B` 停车，记录 Brake 停止距离与冲击。
5. 每轮测试后检查六个马达温度差、异常声音、齿轮和紧固件。

## 5. 需要本机实测后才能调整或解锁的项目

| 项目 | 当前状态 | 实机工作 |
|---|---|---|
| 六端口设备身份与方向 | 操作者提供、未 HIL | 架空逐端口/逐侧确认并记录 |
| 传动比与轮端一致性 | 齿数计算 | 标记转数复核；按轮端归一化比较六电机增量 |
| 控制器零偏/死区 | 候选 `0 / 0.06` | 静止采样并防止中立爬行 |
| 油门/转向曲线 | 线性+15%立方候选 | 驾驶员 A/B 盲测 |
| 输入与输出 Slew | 离线候选 | 检查打滑、电流冲击、换向和急松杆 |
| Brake/Coast/Hold | 当前 Brake | 架空及落地分别验证真实行为 |
| 请求/输出 TTL | 30/40 ms 候选 | 断控、冻结请求、冻结控制发布的 HIL 测试 |
| 电流、温度、卡死阈值 | 未标定 | 冷态/热态/负载重复测试 |
| 实际轮径与有效轮距 | CAD 名义值 | 长直线、CW/CCW 原地旋转拟合 |
| `m_per_motor_rad` | 未标定 | 对每种传动路径做轮端换算后拟合 |
| IMU/位姿/航向辅助 | 无硬件，锁定 | 安装并完成质量、方向、延迟和校准后再实现验收 |
| 自动速度、运动和线路 | 锁定/DoNothing | 完成传感器、几何、闭环、HIL 和路线逐条批准 |

完成端口、方向、停车、断控和 TTL 的正式 HIL 记录后，才允许把 `hardware_output` 升级；完成基础驾驶矩阵后，才允许把 `driver_control` 升级。当前 4 V Test 构建不能直接作为比赛构建。
